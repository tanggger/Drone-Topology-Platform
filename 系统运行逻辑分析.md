# RTK仿真系统运行逻辑分析

## 📋 系统概述

本系统是一个基于ns-3的无人机编队网络仿真平台，从RTK（Real-Time Kinematic）定位数据输入到网络传输和拓扑变化输出的完整流程。

---

## 🔄 完整数据流程图

```
RTK原始数据文件
    ↓
[Python预处理] preprocess.py
    ├─ GPS坐标转换 (WGS84 → ENU)
    ├─ 时间标准化
    ├─ 轨迹插值和平滑
    └─ 输出: processed_trajectories.csv
    ↓
[Python轨迹生成] generate_ns3_traces.py
    ├─ 生成waypoint文件 (uav_*.wp)
    └─ 生成统一trace文件: mobility_trace.txt
    ↓
[ns-3仿真] rtk_benchmark.cc / rtk_simulation.cc
    ├─ 加载轨迹数据
    ├─ 设置移动模型
    ├─ 配置WiFi网络
    ├─ 运行仿真
    └─ 输出结果文件
    ↓
输出文件
    ├─ node-transmissions.csv
    ├─ topology-changes.txt
    ├─ node-positions.csv
    ├─ flowmon.xml
    └─ flow-stats.csv
```

## 📥 第一阶段：RTK数据预处理

### 1.1 输入文件格式

**RTK原始数据** (`rtk_data.csv` 或类似文件):
```csv
drone_id,time_sec,latitude,longitude,altitude
0,0.0,39.123456,116.123456,100.5
0,0.1,39.123457,116.123457,100.6
...
```

### 1.2 预处理流程 (`preprocess.py`)

**主要步骤**:

1. **加载RTK数据** (`load_rtk_data`)
   - 读取CSV文件
   - 解析无人机ID、时间戳、GPS坐标

2. **时间标准化** (`normalize_time`)
   ```python
   sim_time = (time_sec - t0) / time_scale
   ```
   - 将绝对时间转换为相对仿真时间
   - 支持时间缩放（加速/减速仿真）

3. **GPS坐标转换** (`gps_to_enu`)
   ```python
   # WGS84 (经纬度) → ENU (东-北-天，米制)
   x = (longitude - lon0) * lon_to_meter
   y = (latitude - lat0) * lat_to_meter  
   z = altitude - alt0
   ```
   - 选择第一个数据点作为ENU原点
   - 转换为米制坐标系统

4. **轨迹插值** (`interpolate_trajectories`)
   - 将不规则时间间隔的数据插值到固定间隔（默认0.1秒）
   - 使用线性插值确保时间连续性

5. **轨迹平滑** (`smooth_trajectories`)
   - 滑动窗口平均（默认窗口大小5）
   - 减少RTK定位噪声

**输出**: `processed/processed_trajectories.csv`
```csv
drone_id,sim_time,x,y,z
0,0.0,0.328,4.084,18.910
0,0.1,0.330,4.085,18.911
...
```

## 📤 第二阶段：生成ns-3轨迹文件

### 2.1 轨迹文件生成 (`generate_ns3_traces.py`)

**主要功能**:

1. **生成Waypoint文件** (`generate_waypoint_files`)
   - 为每个无人机生成独立的 `.wp` 文件
   - 格式: `time(s) x(m) y(m) z(m)`

2. **生成统一Trace文件** (`generate_trace_file`)
   - 生成 `mobility_trace.txt`
   - 格式: `time(s),nodeId,x(m),y(m),z(m)`

**输出文件示例** (`data_rtk/mobility_trace_cross.txt`):
```
# Mobility trace file
# Format: time(s) nodeId x(m) y(m) z(m)
0.000,14,0.328,4.084,18.910
0.724,8,9.594,-1.465,1.630
3.000,14,0.328,4.007,18.920
...
```

## 🚀 第三阶段：ns-3仿真执行

### 3.1 仿真程序入口

**主程序**: `scratch/rtk_benchmark.cc` (基准测试版本)
**简化版本**: `scratch/rtk_simulation.cc` (基础仿真版本)

### 3.2 仿真初始化流程

#### 步骤1: 加载轨迹数据 (`LoadTrajectoryData`)

```cpp
// 读取mobility_trace.txt
// 解析格式: time,nodeId,x,y,z
// 存储到全局数据结构:
// - g_trajectoryData: 所有轨迹点
// - g_nodeTrajectories: 按节点分组的轨迹
// - g_simulationEndTime: 仿真结束时间
// - g_numNodes: 节点数量
```

**关键处理**:
- 按时间排序
- 移除非严格递增的时间点（避免Waypoint模型错误）
- 确定仿真参数（节点数、时长）

#### 步骤2: 设置移动模型 (`SetupRTKMobility`)

```cpp
// 为每个节点安装WaypointMobilityModel
MobilityHelper mobility;
mobility.SetMobilityModel("ns3::WaypointMobilityModel");
mobility.Install(nodes);

// 为每个节点添加waypoint
for (每个轨迹点) {
    waypoint->AddWaypoint(Waypoint(Seconds(time), Vector(x, y, z)));
}
```

**RTK误差模拟** (仅benchmark版本):
- 基础高斯噪声: `N(0, rtkNoiseStdDev²)`
- 周期性漂移: 模拟RTK信号丢失/恢复

#### 步骤3: 配置WiFi网络

**信道模型**:
```cpp
// ThreeLogDistancePropagationLossModel
// - 三段式路径损耗模型
// - 不同距离使用不同指数

// NakagamiPropagationLossModel  
// - 多径衰落模型
// - 模拟城市环境散射
```

**WiFi参数** (根据难度级别):
- **Easy**: 802.11n, 20MHz, 高功率, 充足重传
- **Moderate**: 802.11n, 20MHz, 中功率, 限制重传
- **Hard**: 802.11n, 20MHz, 低功率, 无重传

#### 步骤4: 设置通信调度 (`ScheduleIntelligentCommunication`)

**智能通信触发机制**:
```cpp
// 每0.1-0.3秒检查一次通信机会
for (t = 1.0; t < endTime; t += step) {
    if (随机数 < commTriggerProb) {
        选择发送节点和接收节点
        计算距离
        根据距离计算通信概率
        if (距离 < maxCommDistance && 概率满足) {
            创建OnOffApplication
            发送TCP/UDP数据包
        }
    }
}
```

**距离相关通信概率**:
```cpp
if (distance <= nearDistance) {
    commProb = 1.0;  // 近距离100%通信
} else {
    commProb = max(0.1, 1.0 - (distance - nearDistance) / decaySpan);
}
```

**业务类型** (benchmark版本):
- **Easy**: 统一TCP数据流
- **Moderate**: TCP传感器数据 + UDP视频流
- **Hard**: 高负载TCP + UDP混合

## 📊 第四阶段：数据采集与输出

### 4.1 传输事件记录 (`Ipv4Tracer`)

**回调函数注册**:
```cpp
Config::Connect("/NodeList/*/$ns3::Ipv4L3Protocol/Tx", MakeCallback(&Ipv4Tracer));
Config::Connect("/NodeList/*/$ns3::Ipv4L3Protocol/Rx", MakeCallback(&Ipv4Tracer));
```

**记录逻辑**:
```cpp
void Ipv4Tracer(context, packet, ipv4, interface) {
    // 1. 解析数据包
    // 2. 过滤TCP协议 (Protocol == 6)
    // 3. 提取节点ID (从context路径)
    // 4. 判断事件类型:
    //    - Tx Data / Tx Ack
    //    - Rx Data / Rx Ack
    // 5. 写入文件: time_s,nodeId,eventType
}
```

**输出文件**: `node-transmissions.csv`
```csv
time_s,nodeId,eventType
1.604,2,Tx Ack
1.613,14,Rx Ack
1.613,14,Tx Ack
...
```

### 4.2 拓扑变化记录 (`TopologyOutput`)

**拓扑链路收集**:
```cpp
// 在Ipv4Tracer中，当检测到数据传输时:
if (peerNodeId != nodeId) {
    uint32_t a = min(nodeId, peerNodeId);
    uint32_t b = max(nodeId, peerNodeId);
    uint32_t idx = floor(当前时间 / 5.0);  // 5秒时间窗口
    g_intervalLinks[idx].insert(pair(a, b));
}
```

**定期输出**:
```cpp
// 每5秒输出一次拓扑快照
Simulator::Schedule(Seconds((idx + 1) * 5.0), &TopologyOutput, idx);

void TopologyOutput(index) {
    // 输出格式: start-end: NodeA-NodeB, NodeC-NodeD, ...
    // 清空该时间窗口的链路集合
}
```

**输出文件**: `topology-changes.txt`
```
0-5: Node0-Node13, Node2-Node14
5-10: Node0-Node11, Node5-Node12, Node7-Node13
10-15: Node3-Node7, Node4-Node7, Node6-Node10, Node10-Node11
```

## 🔍 关键数据流详解

### 5.1 node-transmissions.csv 生成流程

```
数据包传输事件
    ↓
Ipv4L3Protocol::Tx/Rx 回调
    ↓
Ipv4Tracer() 函数
    ├─ 解析数据包头部
    ├─ 过滤TCP协议
    ├─ 提取节点ID和事件类型
    └─ 写入CSV文件
    ↓
node-transmissions.csv
    ├─ time_s: 事件发生时间
    ├─ nodeId: 节点ID
    └─ eventType: Tx Data, Tx Ack, Rx Data, Rx Ack
```

**关键代码位置**:
- `scratch/rtk_benchmark.cc:278-330` - Ipv4Tracer函数
- `scratch/rtk_benchmark.cc:964-965` - 回调注册

### 5.2 topology-changes.txt 生成流程

```
数据传输事件
    ↓
Ipv4Tracer() 检测到有效通信
    ├─ 提取源节点和目的节点
    ├─ 计算时间窗口索引 (当前时间 / 5秒)
    └─ 添加到 g_intervalLinks[窗口索引]
    ↓
每5秒触发 TopologyOutput()
    ├─ 读取该时间窗口的链路集合
    ├─ 格式化输出: start-end: NodeA-NodeB, ...
    └─ 清空该窗口的链路集合
    ↓
topology-changes.txt
    └─ 格式: 0-5: Node0-Node13, Node2-Node14
```

**关键代码位置**:
- `scratch/rtk_benchmark.cc:321-329` - 链路收集
- `scratch/rtk_benchmark.cc:335-355` - TopologyOutput函数
- `scratch/rtk_benchmark.cc:970-974` - 定期调度

## 🎯 运行命令示例

### 完整流程运行

```bash
# 1. 预处理RTK数据
python3 rtk/preprocess.py \
    --input rtk_data.csv \
    --output_dir processed \
    --time_scale 1.0 \
    --dt 0.1

# 2. 生成ns-3轨迹文件
python3 rtk/generate_ns3_traces.py \
    --input processed/processed_trajectories.csv \
    --output_dir data_rtk \

./ns3 run "rtk_benchmark \
    --trajectory=data_rtk/mobility_trace_cross.txt \
    --outputDir=benchmark \
    --formation=cross \
    --difficulty=easy"

# 4. 批量运行所有场景

```

## 📈 输出文件用途

### node-transmissions.csv
- **用途**: 分析网络传输行为
- **指标**: 
  - PDR (Packet Delivery Ratio): RxPackets / TxPackets
  - 传输事件时序分析
  - 节点通信模式识别

### topology-changes.txt
- **用途**: 分析网络拓扑动态性
- **指标**:
  - 拓扑稳定性: Jaccard相似度
  - 链路变化率: 每秒链路变化次数
  - 连通性分析: 节点对连通时间

### node-positions.csv
- **用途**: 位置跟踪和轨迹分析
- **指标**:
  - 位置精度 (RMSE)
  - 编队保持度
  - 空间分布分析

### flowmon.xml / flow-stats.csv
- **用途**: 详细的流级统计
- **指标**:
  - 吞吐量 (Throughput)
  - 时延 (Latency)
  - 抖动 (Jitter)
  - 丢包率 (Packet Loss Rate)

## 📝 总结

### 数据流向
1. **输入**: RTK GPS数据 (经纬度、高度、时间戳)
2. **预处理**: 坐标转换、时间标准化、轨迹平滑
3. **轨迹生成**: ns-3格式的mobility trace文件
4. **仿真执行**: ns-3网络仿真，模拟WiFi通信
5. **数据采集**: 实时记录传输事件和拓扑变化
6. **输出**: CSV和TXT格式的分析数据文件

### 核心机制
- **移动模型**: WaypointMobilityModel，基于RTK轨迹精确移动
- **通信调度**: 基于距离的概率触发机制
- **事件记录**: 通过ns-3回调机制实时捕获网络事件
- **拓扑分析**: 5秒时间窗口的链路快照

### 关键文件
- **输入**: `data_rtk/mobility_trace_*.txt`
- **输出**: 
  - `benchmark/*/node-transmissions.csv`
  - `benchmark/*/topology-changes.txt`
  - `benchmark/*/node-positions.csv`
