## 二、关键发现与分析

### 2.1 可靠性分析（PDR）

**结果：** 所有数据集的 PDR 均为 **1.0000 (100%)**

**分析：**
- ✅ **优秀的可靠性**：无论难度级别和编队类型，所有数据包均成功送达
- 说明 ns-3 仿真中的 WiFi 协议栈（802.11n + 重传机制）工作良好
- 在理想和中等环境下表现优异，即使在困难模式（复杂多径、RTK漂移、高负载）下也能保持完美的投递率
- **实际意义**：算法和协议配置能够应对各种挑战性场景

---

### 2.2 吞吐量分析

#### 按难度分组统计

| 难度级别 | 平均吞吐量(Mbps) | 范围 |
|---------|----------------|------|
| Easy | 0.0259 | 0.0259 - 0.0260 |
| Moderate | 1.0668 | 1.0651 - 1.0693 |
| Hard | 2.7680 | 2.7602 - 2.7773 |

**核心发现：**
1. **难度与吞吐量正相关**：
   - Easy → Moderate: **增长 ~41倍**
   - Moderate → Hard: **增长 ~2.6倍**
   - Easy → Hard: **增长 ~107倍**

2. **原因分析**：
   - Easy 模式：仅心跳包（1 pps × 200字节）+ 极低传感器数据（0.01 Mbps）
   - Moderate 模式：增加中等视频流负载
   - Hard 模式：高负载视频流 + 共信道干扰，触发更多数据传输

3. **编队类型影响较小**：
   - 同难度级别下，不同编队的吞吐量差异 < 1%
   - 说明编队形态对整体吞吐量影响有限

---

### 2.3 时延分析（95分位）

#### 趋势观察

| 难度级别 | 平均95分位时延(ms) | 最佳 | 最差 |
|---------|------------------|------|------|
| Easy | 1.2532 | 1.0913 (v_formation) | 1.3626 (line) |
| Moderate | 0.3389 | 0.2821 (v_formation) | 0.3778 (triangle) |
| Hard | 0.2523 | 0.2366 (v_formation) | 0.2797 (triangle) |

**核心发现：**
1. **反直觉现象**：难度越高，95分位时延**越低**！
   - Easy: ~1.25 ms
   - Moderate: ~0.34 ms（减少 73%）
   - Hard: ~0.25 ms（再减少 26%）

2. **可能解释**：
   - **高负载下队列更活跃**：包不会长时间等待，更快被处理
   - **统计偏差**：高负载下包数量多（Hard模式包数是Easy的~100倍），95分位更准确反映典型时延
   - **简化计算方法**：使用 DelaySum/RxPackets，可能在低包数下受个别异常值影响

3. **编队影响**：
   - **V-Formation 始终最优**：各难度级别均有最低时延
   - **Line 和 Triangle 稍高**：可能与编队拓扑连接性有关

---

### 2.4 抖动分析

#### 按编队和难度分组

| 编队类型 | Easy | Moderate | Hard |
|---------|------|----------|------|
| Cross | 0.44 ms | 3.11 ms | 3.24 ms |
| Line | 0.46 ms | 2.55 ms | **0.42 ms** ✨ |
| Triangle | 0.47 ms | **5.71 ms** ⚠️ | 3.35 ms |
| V-Formation | **0.40 ms** ✅ | 3.63 ms | 0.98 ms |

**核心发现：**
1. **Easy 模式抖动最稳定**（~0.4-0.5 ms）：
   - 理想信道，低负载，时延非常一致

2. **Moderate 模式抖动显著增加**（2.5-5.7 ms）：
   - **Triangle 编队最高（5.71 ms）**：可能因三角拓扑在城市环境下链路质量波动大
   - 周期性 RTK 漂移和视频流负载导致时延不稳定

3. **Hard 模式抖动分化**：
   - Line 和 V-Formation 抖动**回落**（0.42、0.98 ms）✅
   - Cross 和 Triangle 保持较高（3.24、3.35 ms）⚠️
   - **可能原因**：高负载下，Line/V编队的线性/V字拓扑更利于流量均衡

4. **实时应用影响**：
   - Easy 和 Hard(Line/V) 适合低时延敏感应用（语音、控制）
   - Moderate 和 Hard(Cross/Triangle) 需要更大缓冲区

---

### 2.5 拓扑稳定性分析

#### 统计结果

| 难度级别 | 平均稳定性 | 平均链路变化率(次/秒) |
|---------|-----------|-------------------|
| Easy | 0.0457 | 3.33 |
| Moderate | 0.0447 | 3.32 |
| Hard | 0.0622 | 3.90 |

**核心发现：**
1. **整体拓扑非常不稳定**（稳定性 < 0.1）：
   - Jaccard 相似度仅 4-6%，说明相邻时间窗口链路几乎完全不同
   - 链路变化率高达 **3.3-3.9 次/秒**

2. **难度影响**：
   - Easy/Moderate 相似（~0.045，3.3次/秒）
   - Hard 稍高（0.062，3.9次/秒），但提升有限（仅 ~35%）

3. **可能原因**：
   - **高度动态的编队飞行**：无人机持续移动导致链路频繁建立/断开
   - **通信距离阈值**：最大通信距离 200m，近距离 50m，飞行速度快则频繁越界
   - **概率触发机制**：Trigger Probability 0.4，随机性导致链路波动

4. **对协议影响**：
   - 需要快速路由收敛算法
   - 建议使用反应式路由协议（如 AODV）而非主动式（如 OLSR）
   - 需要高效的邻居发现机制

### 2.6 位置跟踪精度分析

#### 按编队分组

| 编队类型 | 平均精度(m) | 最佳 | 最差 |
|---------|-----------|------|------|
| Cross | 1.0795 | 1.0751 (Hard) | 1.0835 (Easy) |
| Line | 1.1091 | 1.1021 (Hard) | 1.1151 (Easy) |
| Triangle | 1.1946 | 1.1876 (Hard) | 1.2008 (Easy) |
| **V-Formation** | **0.7438** ✅ | 0.7432 (Moderate) | 0.7441 (Hard) |

**核心发现：**
1. **V-Formation 精度最高**（~0.74m）：
   - 比其他编队高 **31-38%**
   - 可能因 V 字编队更稳定、位置抖动小

2. **Triangle 精度最低**（~1.19m）：
   - 三角编队可能需要更复杂的协调控制，导致位置抖动增加

3. **难度级别影响极小**：
   - 同编队下，Easy/Moderate/Hard 精度差异 < 1%
   - 说明 RTK 噪声和漂移对位置跟踪精度影响有限（基础噪声 0.01m 已很小）

4. **实际意义**：
   - 所有编队精度均 < 1.2m，满足多数 RTK 应用需求（厘米级至米级）
   - V-Formation 编队更适合高精度定位要求场景

## 四、数据质量说明

### 4.1 计算方法

1. **PDR**: 直接从 flow-stats.csv 计算 `ΣRxPackets / ΣTxPackets`
2. **吞吐量**: 从 flow-stats.csv 读取各流吞吐量并求平均
3. **95分位时延**: 使用 `DelaySum / RxPackets` 还原每包时延分布，取95分位
4. **抖动**: 时延序列的标准差（简化方法）
5. **拓扑稳定性**: 相邻时间窗口链路集合的 Jaccard 相似度
6. **位置精度**: 各节点位置时间序列抖动的 RMSE

### 4.2 局限性

1. **FlowMonitor 解析失败**：
   - XML 文件格式可能与预期不符，回退到简化计算
   - 95分位时延和抖动可能不如直接从 XML 直方图精确

2. **位置精度计算**：
   - 当前使用"位置抖动"作为精度指标（相对指标）
   - 缺少真实理想位置参考（绝对指标）
   - 更适合理解为"位置稳定性"而非绝对跟踪精度

3. **拓扑稳定性**：
   - 当前时间窗口为 5 秒，较粗粒度
   - 更细粒度分析可能发现不同模式

## 五、建议与下一步

### 5.1 进一步分析建议

1. **修复 FlowMonitor 解析**：
   - 检查 flowmon.xml 格式
   - 使用 ns-3 官方工具进行验证

2. **细化时延分析**：
   - 分析平均时延、中位数时延、最大时延
   - 对比不同分位数（50th、75th、99th）

3. **拓扑深度分析**：
   - 计算平均链路寿命
   - 分析节点度分布随时间变化
   - 识别"关键节点"（hub nodes）

4. **相关性分析**：
   - 拓扑稳定性 vs 时延/抖动
   - 位置精度 vs 通信质量
   - 编队类型 vs 各指标的统计显著性检验

### 5.2 仿真改进建议

1. **增加中等负载场景**：
   - 当前 Easy→Moderate→Hard 跳跃较大
   - 建议增加过渡场景（如 0.5 Mbps 视频流）

2. **位置跟踪增强**：
   - 在仿真中记录理想位置（无噪声）
   - 计算绝对位置误差（与理想位置对比）

3. **拓扑稳定性改进**：
   - 测试不同 Trigger Probability（当前 0.4）
   - 测试不同通信距离参数

## 六、结论

### 核心要点

1. ✅ **可靠性卓越**：所有场景 PDR = 100%
2. ⬆️ **吞吐量随负载线性增长**：Easy(0.026) → Moderate(1.07) → Hard(2.77) Mbps
3. 🔄 **时延反直觉**：难度越高，95分位时延反而越低
4. ⚠️ **抖动波动大**：Moderate 模式抖动最高（尤其 Triangle 编队）
5. 📉 **拓扑高度动态**：稳定性仅 4-6%，链路变化率 3.3-3.9 次/秒
6. 🎯 **V-Formation 综合最优**：时延、稳定性、位置精度均领先

### 最终建议

- **通用推荐**：V-Formation 编队在多数指标上表现最佳
- **实时应用**：选择 Line 或 V-Formation + Easy/Hard
- **高吞吐需求**：选择 Hard 模式
- **动态网络研究**：当前所有场景均适合测试动态路由协议
