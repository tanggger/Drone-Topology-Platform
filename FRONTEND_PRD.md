# 翼网全境 (Wing-Net Omni) —— 城市低空物流数字孪生大屏架构设计与接口规范 (PRD)

> **文档状态**：本项目前端已脱离单页面原型的基础试验阶段。本 PRD 面向最终国赛演播与产品化交付，旨在定义一套基于现代前端工程标准（如 Vue/React + WebGL/ECharts）的企业级高维数字孪生大屏架构。

## 📌 一、 项目概况与前端定位

### 1.1 系统与场景定位
本项目前端是专门为 **“城市低空物流快递穿梭”** 场景打造的无人机网络调度沙盘。其首要目的是 **将枯燥的 ns-3 C++ 底层通信图着色调度机制与物理碰撞规避算法，转化为极具视觉冲击力与工程严谨性的专业数字指挥舱** ，向评委全景展示本团队在极限高难度城市基建环境下的保障干预能力。

### 1.2 核心展示与演播演练策略
*   **脱离“伪实时”玩具**：本项目前端明确摒弃仅仅依赖 `Math.random()` 画圆点的敷衍式动画。
*   **视频级录制与稳态推演架构（核心亮点）**：为保证极其稳妥的答辩路演，前端被设计为 **“沙盘布景 -> 底层引擎硬核精算 -> 前端高频无损回放”** 的三段式架构。评委看到的是最终生成的高帧率、无任何卡顿崩溃风险并高度严谨的数据流回放（通过动态抽调不同场景的数据集形成伪实时切换）。

## 🎨 二、 技术栈规划与 UI 视觉规范 (Tech Stack & Design System)

### 2.1 推荐前端技术栈
从单页 HTML 试验走向正规军编制，后续重构标准如下：
*   **外壳框架**：推荐使用 Vue 3 (Vite) 或 React 完成组件化拼装与数据响应。
*   **核心沙盘渲染**：使用 WebGL / Three.js 打造基于高程地块真实的 3D/2.5D (Isometric) 立体物理图景。
*   **仪表与报表库**：引入 Apache ECharts 设计外周极客面板。

### 2.2 视觉语境风格 (Aesthetics)
必须呈现强烈的 **“军用/重工级数字基建指挥舱”** 感：
*   **主题配色**：科幻暗夜底色（Nebula Dark）、毛玻璃面板（Glassmorphism）。
*   **主色调**：青色 (`#00f2ff`) 用于链路畅通状态。
*   **辅色/告警色**：明绿 (`#00ff88`) 标识高可用、亮橙 (`#ffaa00`) 标识物理 NLOS 遮挡链路、血红 (`#ff3b3b`) 作为同频干扰绝命告警重灾区。
*   **字体排印**：使用等宽数字字体（Monospace）渲染跳动的 QoS 数据，以强调极客感。

## 🖥️ 三、 核心大屏组件系统模块划分

### 3.1 核心组件 1：基于场景配置的可视化工具链 (Scenario Editor Builder)
*这是一个给开发者和演播者“录制剧本”专用的造物主工具箱，是前端极为特殊且核心的中控组件。*
*   **动态环境空投**：支持鼠标在主平面（雷达图）上自由拖拽坐标，一键生成代表阻断视网（NLOS）的城市天际线建筑块，或设立带有特定半径范围的超强电磁干扰圈。
*   **跨语言物料输出 (Zero-Code Export)**：将绘制完成的美观坐标，自动矩阵换算并导出为 `scene.json` 文件。这是本前端框架与 NS-3 C++ 底层耦合的“唯一合法协议凭证”。底座据此重编译产出下一段剧本供大屏展示物理避障与通信断带自愈。

### 3.2 核心组件 2：三维物流飞行网络全域沙盘 (Core Sandbox)
*这是占据最大屏幕版图的主展示区，严格回放后端传回的 `positions.csv` 推演切片。*
*   **城建障碍层映射**：高仿真还原导出的 CBD 方块阵列。
*   **微缩立体机匣与队形**：要求前端渲染出能够按 V字/十字 形前进的实体化无人机，并在机匣上配有动态螺旋桨残影。
*   **链路多态染色**：任何一台无人机必须随着空间位移实施绘制连线。无遮挡/健康分配状态为淡青色；穿过大楼边缘或背侧时为橙色虚线；遭受严重同频撞车时必须变粗红线并闪电光。

### 3.3 核心组件 3：深钻多维数据表盘面板 (Interactive Dashboards)
*   **单点聚焦钻取**：单击主沙盘上的任意一个无人机模型，大屏右侧或随动悬浮窗必须瞬间展示其详细生命体征：包括其所属信道 ID、所承载的业务逻辑（高清图传 / 日常控制）以及当前的电池健康消耗百分比。
*   **实时宏观指标折柱**：大屏左右侧边栏部署由 ECharts 驱动的 PDR（投递率）心电图与全网毫秒级延迟仪表盘。

## 📊 四、 核心数据推演协议规范 (Data Transfer Protocol)

本企业级大屏禁止内部伪造算法行为，所有核心运动学、拓扑学表现皆源自读取 C++ 后端在云端或本地预先处理完毕并推演出的标准 JSON 数据层。

### 4.1 回放帧包覆规范
大屏的动画流控制基于帧级状态推演 (Tick-driven Animation)：
```json
{
  "tick": 125,                    // 仿真全局时间戳
  "QoS": {                        // 映射给 ECharts 面板的实时算力结果
    "total_pdr": 0.94,            // 当前机群投递成功率
    "p99_latency_ms": 32.5,       
    "throughput_mbps": 124.2,
    "algo_compute_time_ms": 2.1   // 图着色算法介入耗时
  },
  "uav_nodes": [                  // 遍历渲染 Three.js/Canvas 主舞台的核心数组
    {
      "id": 0, "x": 123.4, "y": 456.7,
      "channel": 1,               // 算法定标的信道染色基值
      "is_conflict": false,       // 是否冲突高亮红线
      "is_nlos": false,           // 航迹是否处于背阴遮断状态
      "energy": 88.5,             // 机电残存量
      "is_active": true           // 飞行姿态保持
    }
  ]
}
```

## 🏃 五、 交互体验与动效平滑准则
作为一款路演级的数字沙盘，其沉浸感的底线要求为：
1.  **引擎级补间动画 (Interpolation)**：若后端日志由于资源限制仅每 1.0 秒输出一帧 JSON 坐标，前端框架绝不允许出现肉眼可见的“帧跳跃跨步”。必须使用贝塞尔曲线补间技术（Tweening），平滑运算并插值出其在当前一秒内的每一个航向偏转姿态。
2.  **剧变预紧感**：当发生如“建筑物割裂机队”、“重大电磁干扰降临”时，不得仅以冰冷数值体现，必须加入 2-3 秒的全局频闪警告特效与抖动滤镜，渲染突发灾难危机感以及算法救场前的真空期张力。
