## 📈 参数设计逻辑

## 🌍 现实场景映射

### Easy 模式：理想测试环境
- **应用场景**: 实验室、测试场、算法验证
- **环境特征**: 开阔空间、无干扰、稳定信号
- **技术条件**: 高精度RTK、优质信道、充足重传
- **性能目标**: 验证算法逻辑正确性

### Moderate 模式：城市应用环境
- **应用场景**: 城市配送、搜救、监控、商业应用
- **环境特征**: 城市建筑、WiFi干扰、多径传播
- **技术条件**: 商用RTK、城市信道、有限重传
- **性能目标**: 平衡性能与可靠性

### Hard 模式：极端恶劣环境
- **应用场景**: 军事作战、灾难救援、极端环境
- **环境特征**: 复杂地形、强电磁干扰、恶劣天气
- **技术条件**: 受限RTK、恶劣信道、无重传
- **性能目标**: 最大化鲁棒性和抗干扰能力

## 🔧 参数调优指南

### 性能优化方向

#### 提升 PDR (降低丢包率)
1. **信道优化**: 降低路径损耗指数，提高Nakagami m值
2. **通信优化**: 增加重传次数，降低数据速率
3. **干扰优化**: 减少干扰节点，降低干扰强度
4. **负载优化**: 降低业务负载，减少通信频率

#### 降低时延
1. **重传优化**: 减少重传次数限制
2. **负载优化**: 降低业务负载和通信频率
3. **信道优化**: 改善信道质量，减少重传需求
4. **干扰优化**: 减少干扰，降低冲突概率

#### 提高拓扑稳定性
1. **定位优化**: 提高RTK精度，减少漂移
2. **信道优化**: 改善信道质量，提高链路稳定性
3. **负载优化**: 降低网络负载，减少拥塞
4. **距离优化**: 调整通信距离，平衡密度与稳定性

### 应用场景适配

#### 城市商业应用
- **参考模式**: Moderate
- **关键参数**: 信道质量、干扰强度、业务负载
- **优化重点**: 平衡性能与可靠性

#### 军用/应急应用
- **参考模式**: Hard
- **关键参数**: 鲁棒性、抗干扰、可靠性
- **优化重点**: 极端环境适应性

#### 测试验证应用
- **参考模式**: Easy
- **关键参数**: 信道质量、定位精度、通信可靠性
- **优化重点**: 算法逻辑验证

## 📝 使用建议

### 算法开发流程
1. **概念验证**: 使用Easy模式验证算法正确性
2. **性能优化**: 使用Moderate模式优化实际性能
3. **鲁棒性测试**: 使用Hard模式测试极端环境适应性
4. **参数调优**: 基于测试结果调整算法参数

### 性能评估流程
1. **基准测试**: 在Easy模式下建立性能基准
2. **场景测试**: 在Moderate和Hard模式下测试场景适应性
3. **对比分析**: 对比不同模式下的性能变化
4. **优化建议**: 基于测试结果提出优化建议

### 应用部署指导
1. **环境评估**: 根据目标应用环境选择合适的模式
2. **参数适配**: 基于实际环境调整仿真参数
3. **性能预测**: 基于仿真结果预测实际性能
4. **风险控制**: 基于Hard模式结果评估风险控制策略

## 🎯 总结

RTK Benchmark 的参数设计体现了以下特点：

1. **系统性**: 覆盖信道、定位、通信、业务、干扰等各个维度
2. **渐进性**: 从理想环境到极端环境的渐进设计
3. **现实性**: 每个参数都对应现实环境特征
4. **平衡性**: 多参数协同产生综合挑战
5. **可调性**: 支持参数调优和场景适配

通过精心设计的参数配置，RTK Benchmark 为无人机编队通信算法提供了全面、真实、可比的测试环境，支持从概念验证到实际部署的全流程开发需求。
