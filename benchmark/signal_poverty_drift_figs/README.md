## 这组图要表达什么

这 5 张图用于说明一个常见但隐蔽的问题：**观测信号贫乏且漂移**。

- **信号贫乏**：节点可用特征主要是弱语义统计量（包数、突发次数、RSSI/PSD 的均值方差、队列均值、重传率等），它们更多反映**负载/尺度/噪声**，而非稳定的“交互规律”。
- **漂移 + 尺度不一致**：同一节点在不同场景/不同时间窗口尺度下，特征分布与轨迹会发生明显变化。
- **后果**：模型容易学到“尺度/噪声模式”（shortcut），而不是可迁移的交互机制，导致 **OOD 泛化差**。

> 说明：本文件夹所有图的数值 **均为编造的合成数据**（synthetic），“形状”服务于叙事表达，用于展示问题本质。

## 文件清单

### 中间部分（3 张）

- **`middle_1_weak_semantic_stats.png`**
  - **含义**：同一节点在两个时间窗口（A/B）下，弱语义统计量呈现明显差异（尤其在 RSSI/PSD、burst、队列、重传等维度）。
  - **想表达**：即使“交互规律”在潜变量层面稳定，观测到的只是噪声统计量；模型很难得到稳定的机制性表征。
  - **坐标含义**
    - **横轴**：特征维度（示例：`tx_packets`, `rx_packets`, `burst_count`, `rssi_mean/std`, `psd_mean/std`, `queue_mean`, `retx_rate`）。
    - **纵轴**：*Normalized statistic*（为了把不同量纲画在一起，做了人为归一化；不是“真实单位”，只看相对高低/差异）。
  - **怎么看**
    - 同一特征上，蓝/橙两根柱子差得越多，说明该特征对**窗口/场景**越敏感（越漂）。
    - 你会看到：计数类（tx/rx/burst）与信道统计（RSSI/PSD）/队列/重传在不同窗口会出现系统性变化。
  - **反映的问题**
    - 节点观测主要是统计量，它不直接携带“谁和谁在交互、交互规则是什么”的语义。
    - 同一节点在不同窗口下就能显著变化 ⇒ 模型容易把这种变化当作可预测线索（shortcut），而非稳定机制。
  - **一句话图注**：*Same node, different windows → weak stats fluctuate; semantics are thin.*

- **`middle_2_correlation_load_dominated.png`**
  - **含义**：特征相关矩阵显示强相关/强耦合（如 tx/rx/burst 与队列等），整体被负载/尺度驱动。
  - **想表达**：高相关意味着模型可通过“负载/尺度签名”走捷径，而非学习交互关系。
  - **坐标含义**
    - **横轴/纵轴**：同一组特征维度（与图1类似）。
    - **颜色**：皮尔逊相关系数 \(r\in[-1,1]\)。红色越深表示正相关越强，蓝色越深表示负相关越强。
    - **数字标注**：仅对 \(|r|\) 较大的格子做了标注（强调强耦合）。
  - **怎么看**
    - 如果很多块呈现大面积深红/深蓝，说明特征之间共享同一个主导因子（通常是“负载/窗口尺度/噪声强度”）。
    - 例如 `tx_packets`、`rx_packets`、`burst_count` 往往高度正相关；RSSI/PSD 的均值与方差也会形成一组强耦合。
  - **反映的问题**
    - 这种“同涨同跌”的结构意味着：模型不需要理解交互，就可以用某个强相关子空间做预测（scale/load shortcut）。
    - 当测试场景的负载/噪声分布变了，这类捷径会失效 ⇒ 泛化差。
  - **一句话图注**：*Weak features are strongly correlated (load-dominated) → shortcut risk.*

- **`middle_3_pca_clusters_by_scale.png`**
  - **含义**：将样本做 PCA 到 2D 后，点云**按窗口尺度（1s/5s/20s）分簇**很明显；而交互标签（marker）高度重叠，难以分离。
  - **想表达**：表示学习更倾向把“尺度差异”编码进 embedding，而非交互类别/规律 → 泛化会崩。
  - **坐标含义**
    - **横轴 PC1 / 纵轴 PC2**：对标准化后的特征做 PCA，取前两主成分（只是“主要变化方向”，没有物理单位）。
    - **每个点**：一个样本（可理解为：某节点在某窗口下的一次观测特征向量）。
    - **颜色**：窗口尺度（1s/5s/20s）。
    - **空心 marker 形状**：交互标签（示意用，代表潜在“交互状态/任务阶段/交互类型”）。
  - **怎么看**
    - 如果点云主要按颜色分开（同色聚一起），说明 embedding 更偏向编码“尺度/统计偏置”。
    - 如果交互标签（marker）在各簇里混在一起，说明交互信号在 embedding 中难以分离（信号薄）。
  - **反映的问题**
    - 表示学习把“窗口尺度差异”当成主要因素 ⇒ 模型会过拟合尺度/噪声模式。
    - 一旦测试时窗口尺度/采样策略/负载变化，embedding 的语义就漂 ⇒ 任务性能崩。
  - **一句话图注**：*Embedding clusters by window scale, not by interaction regime.*

### 右边部分（2 张）

- **`right_1_distribution_drift.png`**
  - **含义**：同一特征（例：RSSI mean）在 Easy/Moderate/Hard 三种场景的分布发生**均值偏移 + 方差变化**。
  - **想表达**：特征分布漂移会让训练/测试不匹配，模型学习到场景签名而不是机制规律。
  - **坐标含义**
    - **横轴**：RSSI 均值（dBm，越靠左通常代表信道更差/干扰更强）。
    - **纵轴**：概率密度（直方图做了 density 归一化）。
    - **彩色直方图**：不同场景（Easy/Moderate/Hard）的分布。
    - **竖线**：各场景的均值位置。
  - **怎么看**
    - 看“峰的位置”是否移动（均值漂移）、“分布宽度”是否变化（方差漂移）。
    - 本图里从 Easy→Hard 明显左移且更分散：同一特征在不同场景完全不是同分布。
  - **反映的问题**
    - 训练集学到的阈值/尺度，在测试场景会失效（典型 covariate shift）。
    - 模型若把 RSSI 分布当作“场景标签”，就会过拟合场景而非交互机制。
  - **一句话图注**：*Same feature, different scenarios → distribution shift (mean/variance).*

- **`right_2_scale_inconsistency_same_node.png`**
  - **含义**：同一节点的“突发率”在不同窗口尺度（1s/5s/20s）下得到的时间序列明显不同（平滑程度/偏置/噪声模式不同）。
  - **想表达**：**同一节点**在不同窗口尺度下“看起来像不同分布”，模型会把尺度当作可预测线索（shortcut），交互规律对模型而言变成非平稳。
  - **坐标含义**
    - **横轴**：时间（秒）。
    - **纵轴**：估计的 burst rate（弱语义统计量；可理解为“单位时间突发/发送事件数”的估计）。
    - **三条曲线**：同一原始过程，用不同窗口尺度（1s/5s/20s）做统计后的结果。
    - **粉色阴影**：人为设置的一段“扰动期”（例如干扰突增/RTK 漂移/策略切换）。
  - **怎么看**
    - 对比三条曲线的形状：1s 更噪，20s 更平滑且带偏置；扰动期在不同尺度下响应幅度和滞后也不同。
    - 这意味着：同一节点“真实状态”不变时，你观察到的特征也会因尺度选择变成不同轨迹。
  - **反映的问题**
    - 多尺度下特征不一致 ⇒ 模型容易把“尺度/平滑程度/噪声纹理”当成判别信息。
    - 从模型视角，交互规律变成“非平稳/不可对齐”的 ⇒ 跨窗口泛化差。
  - **一句话图注**：*Same node, different window scales → inconsistent trajectories.*

## 怎么在论文/答辩里讲这组图（推荐话术）

- **第一句（问题定义）**：我们的节点观测主要是弱语义统计量，缺少直接刻画交互机制的信号。
- **第二句（证据 1：弱语义）**：中间三图说明特征高度统计化、强相关且被尺度主导，embedding 也按尺度分簇。
- **第三句（证据 2：漂移/不一致）**：右侧两图说明场景分布漂移与跨窗口尺度不一致会让特征非平稳。
- **第四句（结论）**：因此模型更容易学习“尺度/噪声模式”，而不是稳定交互规律 → 泛化差。

## 复现与再生成

在本目录执行（会覆盖更新 5 张 PNG）：

```bash
python3 generate_signal_poverty_drift_figs.py
```

如需调整更“夸张”或更“保守”的漂移程度，可在脚本里调这些位置：
- **`synth_feature_vector()`**：控制同一节点两窗口的差异
- **`_synth_dataset_for_correlation()`**：控制负载/尺度/场景对特征的主导程度
- **`plot_right_1_distribution_drift()`**：控制分布漂移幅度
- **`plot_right_2_scale_inconsistency_same_node()`**：控制尺度偏置/扰动强度
