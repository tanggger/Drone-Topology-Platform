# 无人机节点多维特征说明文档

## 📊 概述

本文档详细说明从RTK Benchmark仿真数据中提取的每个无人机节点的多维特征。共提取了 **47个特征维度**，涵盖5个主要类别：

1. **移动性特征** (Mobility Features) - 15个特征
2. **传输特征** (Transmission Features) - 9个特征
3. **流统计特征** (Flow Statistics Features) - 11个特征
4. **拓扑特征** (Topology Features) - 7个特征
5. **性能特征** (Performance Features) - 5个特征

---

## 🚁 一、移动性特征 (Mobility Features)

### 1.1 位置特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `avg_x` | float | 米 | 节点在X轴方向的平均位置 |
| `avg_y` | float | 米 | 节点在Y轴方向的平均位置 |
| `avg_z` | float | 米 | 节点在Z轴方向（高度）的平均位置 |
| `std_x` | float | 米 | X轴位置的标准差（位置变化程度） |
| `std_y` | float | 米 | Y轴位置的标准差 |
| `std_z` | float | 米 | Z轴位置的标准差（高度变化程度） |
| `x_range` | float | 米 | X轴方向的移动范围（最大值-最小值） |
| `y_range` | float | 米 | Y轴方向的移动范围 |
| `z_range` | float | 米 | Z轴方向的移动范围（高度变化范围） |

**应用场景**: 
- 识别节点的空间分布模式
- 分析编队中节点的相对位置
- 评估节点的移动范围

### 1.2 速度特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `avg_velocity_x` | float | m/s | X轴方向的平均速度（绝对值） |
| `avg_velocity_y` | float | m/s | Y轴方向的平均速度（绝对值） |
| `avg_velocity_z` | float | m/s | Z轴方向的平均速度（绝对值） |
| `avg_speed` | float | m/s | 三维空间中的平均速度大小 |
| `max_velocity` | float | m/s | 最大速度（峰值速度） |

**应用场景**:
- 评估节点的移动活跃度
- 识别快速移动的节点
- 分析速度分布模式

### 1.3 移动距离特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `total_distance` | float | 米 | 节点在整个仿真过程中的总移动距离 |
| `avg_step_distance` | float | 米 | 平均每步移动距离 |

**应用场景**:
- 评估节点的移动总量
- 识别移动距离差异大的节点

### 1.4 高度特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `avg_height` | float | 米 | 平均飞行高度 |
| `max_height` | float | 米 | 最大飞行高度 |
| `min_height` | float | 米 | 最小飞行高度 |
| `height_variance` | float | m² | 高度变化的方差 |

**应用场景**:
- 分析飞行高度模式
- 识别高度变化频繁的节点
- 评估编队的垂直分布

### 1.5 位置稳定性

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `position_stability` | float | [0-1] | 位置稳定性指标（值越高越稳定）<br>计算公式: 1/(1+√(Var(x)+Var(y)+Var(z))) |

**应用场景**:
- 识别位置相对固定的节点（如中继节点）
- 评估节点的位置变化程度

## 📡 二、传输特征 (Transmission Features)

### 2.1 数据包传输统计

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `tx_data_count` | int | 个 | 发送的数据包总数 |
| `rx_data_count` | int | 个 | 接收的数据包总数 |
| `tx_ack_count` | int | 个 | 发送的ACK确认包总数 |
| `rx_ack_count` | int | 个 | 接收的ACK确认包总数 |

**应用场景**:
- 评估节点的通信活跃度
- 识别数据源节点和数据汇聚节点
- 分析双向通信模式

### 2.2 传输事件统计

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `total_tx_events` | int | 个 | 总发送事件数（数据包+ACK） |
| `total_rx_events` | int | 个 | 总接收事件数（数据包+ACK） |
| `tx_event_rate` | float | events/s | 发送事件频率 |
| `rx_event_rate` | float | events/s | 接收事件频率 |
| `tx_ratio` | float | [0-1] | 发送/接收比例（0.5表示平衡） |

**应用场景**:
- 评估节点的通信负载
- 识别通信热点节点
- 分析网络流量分布

## 📊 三、流统计特征 (Flow Statistics Features)

### 3.1 作为源节点的特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `flows_as_src` | int | 个 | 作为源节点的流数量 |
| `src_total_tx_packets` | int | 个 | 作为源节点发送的总包数 |
| `src_total_rx_packets` | int | 个 | 作为源节点接收的总包数 |
| `src_avg_pdr` | float | [0-1] | 作为源节点的平均包投递率 |
| `src_avg_throughput` | float | bps | 作为源节点的平均吞吐量 |
| `src_avg_delay` | float | 秒 | 作为源节点的平均时延 |

**应用场景**:
- 识别数据源节点
- 评估源节点的通信质量
- 分析上行链路性能

### 3.2 作为目标节点的特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `flows_as_dst` | int | 个 | 作为目标节点的流数量 |
| `dst_total_rx_packets` | int | 个 | 作为目标节点接收的总包数 |
| `dst_avg_pdr` | float | [0-1] | 作为目标节点的平均包投递率 |
| `dst_avg_throughput` | float | bps | 作为目标节点的平均吞吐量 |

**应用场景**:
- 识别数据汇聚节点
- 评估下行链路性能
- 分析数据接收质量

### 3.3 总体流特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `total_flows` | int | 个 | 节点参与的总流数（作为源+目标） |

**应用场景**:
- 评估节点的网络参与度
- 识别网络中的关键节点

## 🌐 四、拓扑特征 (Topology Features)

### 4.1 邻居特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `num_unique_neighbors` | int | 个 | 在整个仿真过程中出现的唯一邻居节点数量 |

**应用场景**:
- 评估节点的连接多样性
- 识别网络中的枢纽节点

### 4.2 度数特征

| 特征名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| `avg_degree` | float | 个 | 平均度数（平均邻居数） |
| `max_degree` | int | 个 | 最大度数 |
| `min_degree` | int | 个 | 最小度数 |
| `degree_variance` | float | - | 度数的方差 |
| `degree_stability` | float | [0-1] | 度数稳定性（值越高越稳定）<br>计算公式: 1/(1+std(degrees)) |

**应用场景**:
- 评估节点在网络中的重要性
- 识别网络中的中心节点
- 分析拓扑变化模式

## 📈 特征使用建议

### 6.1 特征选择建议

**对于节点分类任务**:
- 移动性特征: `avg_speed`, `position_stability`, `height_variance`
- 传输特征: `tx_ratio`, `total_tx_events`, `tx_event_rate`
- 拓扑特征: `avg_degree`, `num_unique_neighbors`

**对于性能预测任务**:
- 流统计特征: `src_avg_pdr`, `src_avg_throughput`, `src_avg_delay`
- 拓扑特征: `avg_degree`, `degree_stability`
- 移动性特征: `position_stability`, `avg_speed`

**对于异常检测任务**:
- 移动性特征: `max_velocity`, `height_variance`, `position_stability`
- 传输特征: `tx_event_rate`, `rx_event_rate`
- 拓扑特征: `degree_variance`, `degree_stability`

### 6.2 特征工程建议

1. **归一化**: 所有特征应进行归一化处理，特别是不同量纲的特征
2. **特征组合**: 
   - `通信活跃度` = `tx_event_rate` + `rx_event_rate`
   - `移动活跃度` = `avg_speed` × `total_distance`
   - `网络重要性` = `avg_degree` × `total_flows`
3. **时间窗口特征**: 可以按时间窗口提取特征，分析特征的时间演化

### 6.3 特征重要性分析

根据特征类型，建议的重要性排序：

1. **高重要性** (核心特征):
   - `avg_degree` - 网络连接度
   - `src_avg_pdr` - 通信可靠性
   - `position_stability` - 位置稳定性
   - `total_flows` - 网络参与度

2. **中重要性** (辅助特征):
   - `avg_speed` - 移动模式
   - `tx_event_rate` - 通信活跃度
   - `num_unique_neighbors` - 连接多样性

3. **低重要性** (细节特征):
   - `x_range`, `y_range` - 位置范围
   - `height_variance` - 高度变化
   - `tx_ack_count`, `rx_ack_count` - ACK统计

## 🔧 使用方法

### 提取特征

```bash
# 提取单个数据集的特征
python3 extract_node_features.py --dataset cross_Easy --output node_features.csv

# 提取所有数据集的特征
python3 extract_node_features.py --benchmark-dir benchmark --output node_features_all.csv
```

### 加载特征数据

```python
import pandas as pd

# 加载特征数据
df = pd.read_csv('node_features_all.csv')

# 查看特征列表
print(df.columns.tolist())

# 查看某个节点的特征
node_features = df[df['nodeId'] == 0]
print(node_features)

# 按数据集分组
for dataset, group in df.groupby('dataset'):
    print(f"{dataset}: {len(group)} 个节点")
```

### 特征可视化

```python
import matplotlib.pyplot as plt
import seaborn as sns

# 选择关键特征进行可视化
key_features = ['avg_degree', 'src_avg_pdr', 'avg_speed', 'position_stability']

fig, axes = plt.subplots(2, 2, figsize=(12, 10))
for idx, feat in enumerate(key_features):
    ax = axes[idx // 2, idx % 2]
    df.boxplot(column=feat, by='dataset', ax=ax)
    ax.set_title(feat)
    ax.set_xlabel('')
plt.tight_layout()
plt.show()
```

## 📝 数据文件说明

### 输入文件

每个数据集目录包含以下文件，用于特征提取：

1. **node-positions.csv**
   - 格式: `time_s, nodeId, x, y, z`
   - 用途: 提取移动性特征

2. **node-transmissions.csv**
   - 格式: `time_s, nodeId, eventType`
   - 用途: 提取传输特征

3. **flow-stats.csv**
   - 格式: `FlowId, SrcAddr, DestAddr, TxPackets, RxPackets, ...`
   - 用途: 提取流统计特征

4. **topology-changes.txt**
   - 格式: `时间窗口: Node0-Node1, Node2-Node3, ...`
   - 用途: 提取拓扑特征

5. **flowmon.xml**
   - 格式: NS-3 FlowMonitor XML格式
   - 用途: 提取详细性能特征（需要进一步解析）

### 输出文件

- **node_features_all.csv**: 包含所有数据集的所有节点特征
- 格式: CSV格式，每行一个节点，每列一个特征
