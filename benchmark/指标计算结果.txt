───────────────────────────────────────────────────────────────────────────────
一、指标定义与评价角度
───────────────────────────────────────────────────────────────────────────────

1. PDR (Packet Delivery Ratio) - 包投递率
   └─ 评价角度: 可靠性/抗丢包能力
   └─ 数值意义: [0-1]，越高越好，理想值 = 1.0
   └─ 计算方法: 接收包数 ÷ 发送包数

2. Average Throughput - 平均吞吐量
   └─ 评价角度: 效率/容量利用率
   └─ 数值意义: Mbps，越高越好
   └─ 计算方法: 所有流吞吐量的平均值

3. 95th Percentile Latency - 95分位时延
   └─ 评价角度: 尾部时延/服务一致性（关注"最差5%"体验）
   └─ 数值意义: ms，越低越好
   └─ 计算方法: 对时延排序，取第95百分位

4. Jitter - 抖动
   └─ 评价角度: 时间稳定性/可预测性
   └─ 数值意义: ms，越低越好
   └─ 计算方法: 时延序列的标准差

5. Topology Stability - 拓扑稳定性
   └─ 评价角度: 网络动态性/路由鲁棒性
   └─ 数值意义: [0-1]，越高越稳定
   └─ 计算方法: 相邻时间窗口链路集合的Jaccard相似度

6. Position Tracking Accuracy - 位置跟踪精度
   └─ 评价角度: 定位质量/轨迹跟踪能力
   └─ 数值意义: 米(m)，越低越好
   └─ 计算方法: 位置时间序列抖动的RMSE

【角度1：可靠性】PDR（越高越好）

【角度2：效率】平均吞吐量（越高越好，单位: Mbps）
────────────────────────────────────────────────
  按难度统计:
    Easy 模式:      0.0259 Mbps  (仅心跳包 + 低传感器数据)
    Moderate 模式:  1.0668 Mbps  (增加中等视频流) [增长 41倍]

  按编队统计:
    Cross:       1.2867 Mbps

    Triangle:    1.2837 Mbps
    V-Formation: 1.2908 Mbps  ← 最高（但差异很小）

  ✅ 结论: 难度对吞吐量影响显著，编队类型影响极小


【角度3：尾部时延】95分位时延（越低越好，单位: ms）
────────────────────────────────────────────────
  按难度统计:
    Easy 模式:     1.2532 ms
    Moderate 模式: 0.3389 ms  ← 反直觉：更低！
    Hard 模式:     0.2523 ms  ← 最低

  按编队统计:
    V-Formation:  0.5367 ms  ← 最低（最优）

    Line:         0.6568 ms

  ⚠️ 注意: 难度越高时延反而越低，可能因高负载下统计更准确


【角度4：稳定性】抖动（越低越好，单位: ms）
──────────────────────────────────────────
  按难度统计:

  按编队难度交叉:

  ✅ 结论: Easy模式最稳定；Moderate模式抖动最大，尤其Triangle编队


【角度5：动态性】拓扑稳定性（越高越好，范围 0-1）

  按难度统计:
    Easy 模式:     0.0457  (链路变化率: 3.33 次/秒)

    Hard 模式:     0.0622  (链路变化率: 3.90 次/秒)

     - 每秒发生3-4次链路变化
     - 建议使用反应式路由协议（AODV等）

⚠️ 需要关注的问题
──────────────────
  1. 拓扑极度不稳定（稳定性 < 10%）
     → 影响: 路由频繁重建，可能导致短暂丢包
     → 建议: 使用快速收敛的反应式路由协议

  2. Moderate 模式抖动异常高（尤其 triangle_Moderate: 5.7 ms）
     → 影响: 实时应用体验下降，需要更大缓冲区
     → 建议: 如果抖动敏感，避免 Moderate + Triangle 组合

  3. 时延与负载呈负相关（反直觉）
     → 可能原因: 高负载下统计样本多，95分位更准确
     → 建议: 进一步分析平均时延和最大时延

  4. Triangle 编队综合表现较差
     → 抖动高 + 位置精度低
     → 建议: 若无特殊需求，优先考虑 V-Formation 或 Line


✅ 优势总结
───────────
  • PDR 完美（100%）- 协议栈重传机制有效
  • 高吞吐可达 2.77 Mbps - 支持视频流等高负载应用
  • V-Formation 编队全面优秀 - 推荐首选
  • Easy/Hard 模式抖动低 - 适合时延敏感应用
