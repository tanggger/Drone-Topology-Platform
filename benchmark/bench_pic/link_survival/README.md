## 🎯 图表说明

### 坐标轴
- **X轴**: 仿真时间（秒），从0到仿真结束
- **Y轴**: 链路对（Node0-Node1, Node0-Node2, ...）
- **深色条形**: 该链路在此时间段存在
- **空白区域**: 该链路在此时间段不存在

### 颜色编码
- 🟢 **Easy模式**: 绿色条形 (`#2ecc71`)
- 🟠 **Moderate模式**: 橙色条形 (`#f39c12`)
- 🔴 **Hard模式**: 红色条形 (`#e74c3c`)

### 排序规则
链路按**活跃度**（总存活时长）降序排列：
- 顶部：最活跃的链路（存在时间最长）
- 底部：偶尔出现的链路（存在时间最短）

## 🔍 如何解读图表

### 1. 横向密度 → 链路稳定性

#### 高密度（连续长条）
```
Node5-Node8  ████████████████████████████  ← 稳定链路
```
**含义**：
- ✅ 链路长时间保持连接
- ✅ 通信可靠，无频繁断连
- ✅ 环境理想，信号强度稳定
- ✅ 节点相对位置变化小

#### 低密度（破碎短条）
```
Node3-Node12  ██  ███    █  ███  █████    ← 不稳定链路
```
**含义**：
- ⚠️ 链路频繁断开重连
- ⚠️ 通信不稳定，需要频繁重传
- ⚠️ 环境恶劣，干扰或衰落严重
- ⚠️ 节点快速移动，频繁进出通信范围

---

### 2. 纵向密度 → 网络连接性

#### 高密度（很多链路同时存在）
**含义**：
- ✅ 网络连接性强，多路径可选
- ✅ 容错能力高，单链路失效影响小
- ✅ 编队紧凑，节点间距离近
- ✅ 适合多跳路由和冗余传输

#### 低密度（只有少数链路）
**含义**：
- ⚠️ 网络稀疏，可能出现分割
- ⚠️ 容错能力差，关键链路失效影响大
- ⚠️ 编队分散，节点距离远
- ⚠️ 路由选择受限

---

### 3. 整体密度 → 场景难度

#### Easy 模式（预期：高密度）
- 大面积深绿色
- 长连续条形为主
- 60-80条链路同时活跃
- 像"满满当当"的日程表

#### Moderate 模式（预期：中等密度）
- 橙色条形开始破碎
- 仍有主干链路保持稳定
- 40-60条链路活跃
- 像"偶尔有空隙"的日程表

#### Hard 模式（预期：低密度）
- 红色条形高度碎片化
- 像"锯齿状"或"心电图"
- 20-40条链路活跃
- 像"满是空白"的日程表

## 💡 观察要点

### 对比 Easy vs Hard
1. **打开同一编队的 Easy 和 Hard 图**
2. **观察密度变化**：
   - Easy 应该"满满当当"（高密度绿色）
   - Hard 应该"千疮百孔"（低密度红色）
3. **如果差异不明显**：说明难度设置可能不够极端

### 寻找骨干链路
1. **查看顶部20%的链路**
2. **如果顶部明显比底部密集**：说明存在"核心链路"承担主要通信
3. **如果密度均匀**：说明链路利用较平衡

### 发现网络事件
1. **垂直空洞**（某个时间段整体空白）：
   - 可能发生网络分割
   - 或全局性干扰事件
2. **水平空白**（某些链路始终空白）：
   - 这些节点对从未接近过
   - 编队设计导致某些节点无法直接通信

## 🔬 技术细节

### 数据来源
- **文件**: `topology-changes.txt`（每个数据集）
- **格式**: 
  ```
  0-5: Node0-Node13, Node2-Node14
  5-10: Node0-Node11, Node5-Node12
  ```

### 处理逻辑
1. 解析时间区间和节点对
2. 合并重叠的时间区间
3. 按总存活时长排序
4. 分面绘制（每40条链路一个子图）

### 绘图参数
- **分辨率**: 150 DPI
- **尺寸**: 16×(8-20)英寸（根据链路数自适应）
- **工具**: Matplotlib `barh`
